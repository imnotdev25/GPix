Quick guide

Turn an endpoint on by adding its *_listen line to gpix-web.conf and restarting gpix. Then use the generated credentials:

{{if $g.S3Enabled}}

S3 (AWS CLI)

export AWS_ACCESS_KEY_ID={{if $g.HasS3Keys}}{{$g.S3AccessKey}}{{else}}<access-key>{{end}}
export AWS_SECRET_ACCESS_KEY=<your-secret>
aws --endpoint-url {{$g.S3Endpoint}} s3 ls s3://{{$g.S3Bucket}}/
aws --endpoint-url {{$g.S3Endpoint}} s3 cp ./report.pdf s3://{{$g.S3Bucket}}/
aws --endpoint-url {{$g.S3Endpoint}} s3 cp s3://{{$g.S3Bucket}}/report.pdf ./out.pdf
{{end}} {{if $g.WebDAVEnabled}}

WebDAV (rclone)

rclone config create gpix webdav url {{$g.WebDAVEndpoint}} vendor other \
  user {{$g.WebDAVUsername}} pass <app-password-or-login-password>
rclone ls gpix:
rclone copy ./report.pdf gpix:

WebDAV (curl)

curl -u {{$g.WebDAVUsername}}:<password> -T report.pdf {{$g.WebDAVEndpoint}}/report.pdf
curl -u {{$g.WebDAVUsername}}:<password> {{$g.WebDAVEndpoint}}/report.pdf -o out.pdf
{{end}}

macOS Finder: Go → Connect to Server → {{if $g.WebDAVEndpoint}}{{$g.WebDAVEndpoint}}{{else}}http://host:8081{{end}}. Windows: Map network drive with the same URL.
